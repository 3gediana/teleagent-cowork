# 可进化的平台 (Evolvable Platform)

> 本文档记录平台演进的核心方向：让平台通过数据积累变得越来越聪明。
> 
> 目前处于"愿景阶段"，具体实现方案待定。

---

## 1. 核心理念

**平台不只是一个工具，而是一个会学习的系统。**

每一次有人使用平台，都应该让平台变得更好一点点。

---

## 2. 数据收集原则（待细化）

- [ ] 收集什么数据？（交互日志、执行结果、决策过程）
- [ ] 怎么收集才不影响用户体验？
- [ ] 什么数据是"好的数据"？怎么筛选？
- [ ] 用户隐私怎么处理？

> 核心问题：不是所有数据都有用，需要设计收集机制和筛选机制

---

## 3. 平台能积累什么（愿景）

| 类型 | 说明 |
|------|------|
| **任务模式** | 什么任务对应什么代码模式 |
| **审核规律** | 什么样的改动会被拒绝，为什么 |
| **规划模式** | 什么样的项目规划能成功，什么样的会返工 |
| **冲突模式** | 什么情况下会产生问题 |
| **依赖关系** | 任务之间的依赖关系 |
| **架构模式** | 好的代码结构长什么样 |

---

## 4. 分析机制（愿景）

**派专门的Agent来做统计分析**：
- 不是实时的，是在后台跑
- 从大量数据中提炼规律
- 把规律凝练成可用的"经验"
- 经验不是写死的文档，而是平台的判断依据

---

## 5. 经验怎么用（愿景）

| 场景 | 用法 |
|------|------|
| 新项目规划时 | 告诉人类："参考历史，这个方向成功率高" |
| AI接任务时 | 警告："这个任务可能遇到这些问题" |
| 审核时 | 告诉AI："这个项目历史上40%的拒绝是因为这个原因" |
| 架构设计时 | 建议："参考类似项目的架构" |

---

## 6. 关键词

```
数据收集 → 规律分析 → 经验凝练 → 应用场景
```

---

## 7. 待解决的核心问题

1. **什么算"好的数据"？** — 不是所有交互都有价值，需要筛选
2. **怎么从数据里提炼规律？** — 需要统计分析Agent
3. **经验怎么存储？** — 不是写死的文档，是动态的知识
4. **怎么让经验变成平台的判断？** — 需要反馈闭环
5. **用户隐私怎么保护？** — 数据脱敏？匿名化？

---

## 8. 不需要现在解决，但需要留好接口

当前阶段（Phase 1/2）应该：
- 设计好数据结构，留好字段
- 记录每一次关键交互
- 不要急着做分析，先积累数据

等到数据量足够、平台稳定之后，再回过头来做Phase 3。
