## 十、待讨论事项

### 10.1 GitHub 集成 ✅ 已解决

| 问题 | 解决方案 |
|------|----------|
| **代码存储** | 平台本地存储 Git 仓库，用户通过 MCP 工具同步 |
| **用户同步** | 用户调用 file.sync 获取差异文件，MCP 工具自动处理 |
| **批量提交** | 平台空闲时批量 push 到 GitHub，减少 API 调用 |
| **避免频繁操作** | 减少 GitHub push/pull 操作，提高稳定性 |

**用户端文件同步流程**：

```
用户调用 file.sync({ version: "v1.2.3" })
    ↓
MCP 工具读取本地 .a3c_version
    ↓
发送版本号给平台
    ↓
平台对比差异，返回操作列表
    ↓
MCP 工具自动执行文件操作（覆盖/删除）
    ↓
用户端保存版本号到 .a3c_version
```

**冲突检测**：
```
用户本地有修改 + 再次 sync
    ↓
MCP 工具计算本地文件 hash
    ↓
平台检测 hash 不匹配
    ↓
返回错误：本地有修改，请先 change.submit
```

**平台 Git 提交流程**：

```
审核 Agent 审核通过
    ↓
git add && git commit（生成新版本号）
    ↓
平台空闲时（无用户在线 > N 分钟）
    ↓
git push 到 GitHub
```

**提交时机**：
| 情况 | 触发 |
|------|------|
| **空闲提交** | 无用户在线超过 N 分钟 |
| **定时提交** | 每小时检查一次 |
| **手动提交** | 项目管理员触发 |

### 10.2 审核 Agent 实现 ✅ 已解决

| 问题 | 解决方案 |
|------|----------|
| **双Agent协作** | 审核 Agent1 首审 + 修复 Agent 验证 + 审核 Agent2 复审（误判时） |
| **审核等级** | L0无冲突直接合并，L1转修复Agent，L2打回用户端 |
| **版本过期** | 规则层拦截，MCP直接返回"请同步后重试"，不进审核队列 |
| **冲突分类** | 物理冲突（规则层）、依赖冲突（静态分析）、语义冲突（AI判断） |
| **输入设计** | 系统注入（diff、任务详情、方向块）+ AI主动读取（提交文件、仓库代码） |
| **打回路经** | 审核Agent→用户端，修复Agent→审核Agent2/用户端 |

**审核流程**：
```
1. 规则层检查版本，过期直接返回
2. 审核 Agent1 判断等级（L0/L1/L2）
3. L0 → 合并，L2 → 打回用户端
4. L1 → 修复Agent验证
   - 确认问题存在 → 修复 → 合并
   - 发现误判 → 审核 Agent2 复审
   - 发现涉及其他文件 → 打回用户端
```

### 10.12 回滚机制 ✅ 已解决

| 问题 | 解决方案 |
|------|----------|
| 回滚范围 | 只做代码回滚（git revert），不做状态回滚 |
| 版本号 | 回滚后版本号退回旧版本号（如v1.3.0→v1.2.0） |
| 触发方式 | 人类在看板指示回滚版本 → 维护Agent生成回滚指令 → 人类确认 → 平台执行 |
| 用户端同步 | 广播VERSION_ROLLBACK后，用户端file.sync拉取旧版本 |
| 状态不回滚理由 | 任务已完成/锁已释放/里程碑已切换，回滚过于复杂且与确认机制冲突 |

### 10.9 任务系统设计 ✅ 已解决

**设计决策**：任务创建采用中心化模式，仅由平台端维护Agent负责。

| 问题 | 解决方案 |
|------|----------|
| **任务创建权** | 仅平台端维护Agent可创建，用户端不能创建 |
| **用户端职责** | 只能领取任务（task.claim）和完成任务（task.complete） |
| **任务来源** | 人类看板输入、维护Agent自动创建、审核通过后分析 |

**自动创建任务开关**：

| 开关状态 | 行为 |
|---------|------|
| **关闭** | 任务做完就停，等人类在看板发布新任务 |
| **打开** | 任务少于N个时，维护Agent自动判断是否需要创建新任务 |

**工具变更**：
- 删除 `task.create` 工具
- 合并 `task.claim` + `task.complete` 为 `task` 工具
