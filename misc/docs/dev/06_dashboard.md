## 2. 页面流程

### 2.1 首次进入

1. 显示方向块、里程碑块、任务列表
2. 对话区显示欢迎信息
3. 人类可以直接输入或选择目标块开始对话

### 2.2 对话-对齐-确认循环

```
人类选择目标块 + 输入自然语言
    ↓
维护Agent接收，多轮对话对齐需求
    ↓
维护Agent引导细化，逐步把模糊需求结构化
    ↓
人类确认"对齐完成" → 点击确认更新按钮
    ↓
如果目标块是方向块 → 需要二次确认
    ↓
维护Agent一次性更新对应块
    ↓
SSE推送更新到左侧面板 + 广播所有Agent
    ↓
人类点击"清空上下文"，开始下一次对齐
```

### 2.3 里程碑切换交互

```
维护Agent提议: "当前里程碑所有任务已完成，建议切换到下一里程碑"
    ↓
人类在看板确认
    ↓
平台执行归档+清空+新建
    ↓
维护Agent填写新里程碑内容
    ↓
广播通知所有Agent
```

### 2.4 代码回滚交互

```
人类在看板输入: "回滚到 v1.2.0"
    ↓
维护Agent生成回滚指令（确认回滚版本号、影响范围）
    ↓
人类确认
    ↓
平台执行 git revert
    ↓
广播 VERSION_ROLLBACK
```

## 3. SSE实时更新

### 3.1 事件类型

| 事件 | 更新的面板区域 | 说明 |
|------|---------------|------|
| DIRECTION_CHANGE | 方向块卡片 | 方向内容更新 |
| MILESTONE_UPDATE | 里程碑块卡片 | 里程碑内容/任务更新 |
| MILESTONE_SWITCH | 里程碑块卡片 | 里程碑切换 |
| VERSION_UPDATE | 版本卡片 | 版本号更新 |
| VERSION_ROLLBACK | 版本卡片 | 版本回滚 |
| 任务变更 | 里程碑块卡片 | 任务状态变更 |
| 锁变更 | 文件锁卡片 | 锁定/释放 |
| Agent上线/下线 | Agent列表 | 在线状态 |

### 3.2 更新策略

- 所有块状态变更通过SSE实时推送到看板
- 看板前端监听SSE事件，自动更新对应卡片
- 不需要手动刷新

---

## 4. 看板输入类型

| 目标块 | 输入内容 | 处理方式 | 确认要求 |
|--------|----------|----------|----------|
| 方向块 | 项目方向、目标变更 | 维护Agent生成proposed_direction | 二次确认 |
| 里程碑块 | 阶段目标、任务需求 | 维护Agent直接更新 | 一次确认 |
| 任务块 | 具体任务描述 | 维护Agent创建任务 | 一次确认 |
