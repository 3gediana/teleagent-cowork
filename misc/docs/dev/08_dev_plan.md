## 2. 里程碑划分

### Phase 1: 基础设施（M01 + M08 + MCP Server骨架）

**目标**：项目骨架 + 数据库 + 认证 + Git操作 + MCP Server骨架

| 任务 | 说明 | 验收标准 |
|------|------|----------|
| Go项目骨架 | Go项目初始化、配置、日志 | 服务能启动，健康检查通过 |
| 数据库 | MySQL建表、迁移脚本 | 所有表创建成功，索引正确 |
| Redis | 连接、基础Key操作 | 读写正常 |
| 项目创建 | Web界面创建/导入项目 | 空项目创建、已有项目导入 |
| Agent注册 | Web界面注册 + 密钥生成（明文10+位随机串） | 注册成功，密钥可登录 |
| 登录/登出 | 认证、session、项目选择 | 登录返回项目上下文 |
| 心跳 | 心跳续租、断线检测 | 5分钟无心跳自动释放 |
| Git操作 | 本地仓库初始化、commit、diff、revert | 基本Git操作正常 |
| 版本号 | 版本号生成、存储 | 版本递增正确 |
| MCP Server骨架 | TypeScript项目初始化、MCP连接、7个工具壳 | OpenCode能连接MCP Server |

**预估代码量**：4000-5000行（Go后端3000-4000 + MCP Server 1000）

### Phase 2: 核心功能（M02 + M03 + M04）

**目标**：任务、锁、改动提交的核心流程

| 任务 | 说明 | 验收标准 |
|------|------|----------|
| 任务创建 | 维护Agent创建任务 | 任务归入当前里程碑 |
| 任务领取 | Agent领取任务 | 防止重复领取 |
| 任务完成 | Agent完成任务 | 锁自动释放 |
| 任务删除 | 维护Agent删除任务 | 状态标记为deleted |
| 文件锁 | 锁定/释放/追加/续租 | 同一文件不可重复锁 |
| 锁TTL | 5分钟过期+心跳续租 | 过期自动释放 |
| 改动提交 | 文件上传+版本检查 | 版本过期拒绝 |
| 审核队列 | FIFO排序 | 先进先审 |
| 状态同步 | 查询任务和锁状态 | 返回最新数据 |
| MCP工具实现 | 7个MCP工具连接平台API | OpenCode能调用工具操作平台 |

**预估代码量**：3000-4000行

## 4. 风险与待定项

| 风险 | 说明 | 应对策略 |
|------|------|----------|
| OpenCode集成复杂度 | 无头模式、session管理、工具注入 | 提前做技术验证 |
| Agent prompt质量 | AI理解精度影响审核结果 | 迭代优化prompt |
| 并发锁冲突 | 5人同时操作可能冲突 | 锁机制+重试策略 |
| SSE稳定性 | 长连接可能断线 | 自动重连+轮询兜底 |
| 开源风险 | MCP协议版本兼容性 | 跟进@modelcontextprotocol/sdk最新版本 |

### 待定项（技术细节待讨论）

| 项目 | 说明 |
|------|------|
| Go Web框架选型 | Gin vs Fiber |
| ORM选型 | GORM vs sqlx |
| MCP Server框架 | @modelcontextprotocol/sdk (TypeScript) |
| MCP Server传输 | remote (HTTP) 或 local (stdio) |
| 前端状态管理 | Zustand vs Jotai |
| 前端UI组件库 | shadcn/ui vs Ant Design |
| CORS策略 | 具体允许的域名 |
| 限流规则 | 具体QPS限制 |
| Agent prompt具体措辞 | 需要在开发中迭代 |
| MCP工具具体实现 | TypeScript MCP Server，7个工具+后台轮询线程 |
| MCP连接方式 | remote类型（HTTP），用户配置opencode.json |
| Web框架的错误处理 | 统一错误格式 |
| 日志规范 | 具体日志级别和格式 |
| 测试策略 | 单元测试覆盖率和集成测试方案 |
| 里程碑切换原子性 | 归档+清空+新建需要事务保证 |
| 评估Agent输出解析 | ASSESS_DOC.md解析成里程碑块的映射规则 |
