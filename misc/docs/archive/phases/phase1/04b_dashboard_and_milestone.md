## 六-B、看板与里程碑

### 6.B.1 看板分层模型

平台提供三类看板，对应不同层级：

| 看板类型 | 对应块 | 用途 |
|----------|--------|------|
| 方向看板 | direction | 定义项目目标、范围、约束 |
| 里程碑看板 | milestone | 定义阶段目标与执行路径 |
| 任务看板 | task | 定义具体执行任务 |

说明：
- 用户必须在对应层级的看板中表达需求
- 不同看板输入不会自动跨层级生效

### 6.B.2 看板界面设计

看板是Web仪表盘的一部分，作为人类干预项目的唯一入口。

**核心交互模式**：看板不是"填入→解析"，而是**对话→对齐→确认**的循环过程。维护Agent通过多轮对话引导人类逐步细化需求，直到颗粒度足够后再一次性更新对应块。

**页面布局**：

```
┌─────────────────────────────────────────────────────────────┐
│                    A3C 仪表盘 (Web)                          │
├──────────────────────┬──────────────────────────────────────┤
│                      │                                      │
│   左侧：块状态展示    │   右侧：对话区                       │
│                      │                                      │
│  ┌────────────────┐  │  ┌────────────────────────────────┐  │
│  │ 方向块         │  │  │ [对话历史：多轮对齐过程]        │  │
│  │ (当前内容)     │  │  │                                │  │
│  └────────────────┘  │  │ ...                            │  │
│                      │  │                                │  │
│  ┌────────────────┐  │  └────────────────────────────────┘  │
│  │ 里程碑块       │  │  [目标块选择器 ▼]  [确认更新] [清空] │
│  │ (当前内容)     │  │  ┌────────────────────────────────┐  │
│  └────────────────┘  │  │ 输入框                         │  │
│                      │  └────────────────────────────────┘  │
│  ┌────────────────┐  │                                      │
│  │ 任务块         │  │                                      │
│  │ (当前任务列表) │  │                                      │
│  └────────────────┘  │                                      │
│                      │                                      │
│  ┌────────────────┐  │                                      │
│  │ 文件锁块       │  │                                      │
│  │ (锁定状态)     │  │                                      │
│  └────────────────┘  │                                      │
│                      │                                      │
│  ┌────────────────┐  │                                      │
│  │ 版本块         │  │                                      │
│  │ (当前版本)     │  │                                      │
│  └────────────────┘  │                                      │
│                      │                                      │
├──────────────────────┴──────────────────────────────────────┤
│  其他块（按需展示）                                           │
└─────────────────────────────────────────────────────────────┘
```

**交互要素**：

| 元素 | 说明 |
|------|------|
| 对话区 | 单一对话区，维护Agent与人类多轮对话对齐需求 |
| 目标块选择器 | 下拉选择当前讨论的目标块（方向块/里程碑块/任务块） |
| 确认更新 | 人类确认对齐完成，维护Agent一次性更新对应块 |
| 清空上下文 | 人类手动点击清空对话上下文，开始下一次对齐 |

**交互流程**：

```
人类在对话区输入自然语言
    ↓
选择目标块（方向块/里程碑块/任务块）
    ↓
维护Agent接收，维护Agent与人类多轮对话对齐需求
    ↓
对齐过程中：维护Agent引导细化，逐步把模糊需求结构化
    ↓
人类确认对齐完成 → 点击"确认更新"
    ↓
维护Agent一次性更新对应块（方向块需二次确认）
    ↓
通过SSE实时推送更新到左侧块状态
    ↓
人类点击"清空上下文"，开始下一次对齐
```

**上下文管理**：

- 单一共享上下文，所有目标块共用
- 对齐完成 + 确认更新后，由人类手动点击"清空上下文"按钮清理
- 不自动清理，维护Agent上下文由人类完全掌控

### 6.B.3 里程碑生命周期

里程碑是项目执行的阶段性划分，与任务形成层级关系：里程碑 → 任务。

**角色分工**：

| 角色 | 职责 |
|------|------|
| 维护Agent | 在里程碑块中描述里程碑内容，提议切换 |
| 平台 | 执行归档、清空、创建等系统操作 |
| 人类 | 确认是否切换里程碑（决策权） |

**生命周期流程**：

```
项目启动
    ↓
人类看板定义方向 → 维护Agent填写第一个里程碑内容
    ↓
┌─────────────────────────────────────┐
│  里程碑执行阶段                       │
│  - 维护Agent创建任务（归入当前里程碑）  │
│  - 用户端领取任务、锁文件、改代码       │
│  - 审核通过 → 任务完成                 │
│  - 维护Agent更新里程碑块              │
└─────────────────────────────────────┘
    ↓
当前里程碑所有任务完成
    ↓
维护Agent判断：
├── 还需要任务 → 自动创建（开关打开时）
└── 里程碑完成 → 提议切换
        ↓
    人类确认？
    ├── 是 → 平台执行切换
    └── 否 → 人类手动加任务
```

**切换操作（平台执行）**：

```
1. 归档当前里程碑
   - 保存里程碑名、描述、方向块快照
   - 保存所有任务记录（含完成者、改动文件、审核次数）
   - 保存版本号
   - 写入 milestone_archive 表
   
2. 清空当前数据
   - 清空任务表（已归档）
   - 标记当前里程碑为 completed
   
3. 创建新里程碑
   - milestone 表新增记录，status = in_progress
   - 维护Agent填写新里程碑内容到里程碑块
   
4. 广播通知
   - 通知所有Agent：新里程碑已开始
```

**归档内容**（给人类回顾AI工作）：

| 字段 | 说明 |
|------|------|
| 里程碑名 | 本次里程碑的名称和目标 |
| 任务列表 | 每个任务的完成者、耗时、审核次数 |
| 版本范围 | 起止版本号 |
| 方向快照 | 归档时的方向块内容 |
